# Режим: otliknutsya -- Живой ассистент для отклика

Интерактивный режим для случаев, когда кандидат заполняет форму отклика в Chrome. Читает содержимое экрана, загружает контекст предыдущей оценки вакансии и генерирует персонализированные ответы на каждый вопрос формы.

## Требования

- **Лучше всего с Playwright в видимом режиме**: Кандидат видит браузер, Claude может взаимодействовать со страницей.
- **Без Playwright**: кандидат делится скриншотом или вставляет вопросы вручную.

## Workflow

```
1. ОБНАРУЖИТЬ  → Прочитать активную вкладку Chrome (скриншот/URL/заголовок)
2. ОПРЕДЕЛИТЬ  → Извлечь компанию + роль со страницы
3. НАЙТИ       → Сопоставить с существующими отчётами в reports/
4. ЗАГРУЗИТЬ   → Прочитать полный отчёт + Блок G (если есть)
5. СРАВНИТЬ    → Роль на экране совпадает с оценённой? Если изменилась → предупредить
6. РАЗОБРАТЬ   → Определить ВСЕ видимые вопросы формы
7. СГЕНЕРИРОВАТЬ → Для каждого вопроса создать персонализированный ответ
8. ПОКАЗАТЬ    → Вывести ответы, отформатированные для copy-paste
```

## Шаг 1 -- Обнаружить вакансию

**С Playwright:** Сделать snapshot активной страницы. Прочитать заголовок, URL и видимое содержимое.

**Без Playwright:** Попросить кандидата:
- Поделиться скриншотом формы (Read tool читает изображения)
- Или вставить вопросы формы как текст
- Или назвать компанию + роль, чтобы мы нашли контекст

## Шаг 2 -- Определить и найти контекст

1. Извлечь название компании и заголовок роли со страницы
2. Искать в `reports/` по названию компании (Grep case-insensitive)
3. Если есть match → загрузить полный report
4. Если есть Блок G → загрузить черновики ответов как базу
5. Если НЕТ match → предупредить и предложить запустить быструю auto-pipeline

## Шаг 3 -- Обнаружить изменения в вакансии

Если роль на экране отличается от оценённой:
- **Предупредить кандидата**: "Роль изменилась с [X] на [Y]. Хотите, чтобы я переоценил или адаптирую ответы под новый заголовок?"
- **Если адаптировать**: Скорректировать ответы под новый заголовок без переоценки
- **Если переоценить**: Выполнить полную оценку A-F, обновить report, пересоздать Блок G
- **Обновить tracker**: Изменить заголовок роли в `applications.md` если нужно

## Шаг 4 -- Разобрать вопросы формы

Определить ВСЕ видимые вопросы:
- Текстовые поля (сопроводительное письмо, почему эта роль, мотивация и т.д.)
- Выпадающие списки (как узнали о вакансии, разрешение на работу и т.д.)
- Да/Нет (готовность к переезду, виза, доступность и т.д.)
- Поля зарплаты (диапазон, ожидания)
- Поля загрузки (резюме, сопроводительное письмо в PDF)

Классифицировать каждый вопрос:
- **Уже отвечен в Блоке G** → адаптировать существующий ответ
- **Новый вопрос** → сгенерировать ответ из report + `cv.md`

## Шаг 5 -- Сгенерировать ответы

Для каждого вопроса генерировать ответ, следуя:

1. **Контекст из report**: Использовать proof points из Блока B, истории STAR из Блока F
2. **Предыдущий Блок G**: Если есть черновик — использовать как базу и доработать
3. **Тон "Я выбираю вас"**: Тот же framework, что в auto-pipeline — уверенный, не умоляющий
4. **Конкретика**: Ссылаться на что-то конкретное из JD, видимого на экране
5. **career-ops proof point**: Включить в "Дополнительная информация" если есть поле для этого

**Поля, специфичные для российского рынка:**
- **Зарплатные ожидания (gross, в месяц или в год)** → Диапазон из `profile.yml`, в рублях, с пометкой "обсуждается в зависимости от полного пакета"
- **Формат трудоустройства (КТК/ГПХ/ИП)** → Ответить согласно `profile.yml`, или "открыт к обоим вариантам" если применимо
- **Готовность к переезду / релокации** → Честно, согласно `profile.yml`
- **Разрешение на работу** → Если гражданин РФ: "Гражданин РФ, разрешение на работу не требуется"
- **Языки** → Указать уровень по каждому языку (родной, свободный, средний, начальный)
- **Доступность / срок выхода** → Реалистичная дата с учётом текущей отработки (КТК: стандартно 2 недели, до 1 месяца)

**Формат вывода:**

```
## Ответы для [Компания] -- [Роль]

Основа: Report #NNN | Score: X.X/5 | Архетип: [тип]

---

### 1. [Точный вопрос из формы]
> [Ответ, готовый для copy-paste]

### 2. [Следующий вопрос]
> [Ответ]

...

---

Заметки:
- [Любые наблюдения о вакансии, изменениях и т.д.]
- [Предложения по персонализации, которые кандидат должен проверить]
```

## Шаг 6 -- После отклика (опционально)

Если кандидат подтвердил отправку отклика:
1. Обновить статус в `applications.md` с "Evaluated" на "Applied"
2. Обновить Блок G в report финальными ответами
3. Предложить следующий шаг: `/career-ops contacto` для LinkedIn outreach

## Scroll handling

Если в форме больше вопросов, чем видно на экране:
- Попросить кандидата прокрутить и поделиться ещё одним скриншотом
- Или вставить оставшиеся вопросы
- Обрабатывать итерациями, пока не покроем всю форму
