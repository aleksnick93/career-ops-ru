# career-ops — Русские моды (`modes/ru/`)

Эта папка содержит русскоязычные переводы ключевых режимов career-ops для кандидатов, которые ищут работу на российском рынке или работают с русскоязычными вакансиями.

## Когда использовать эти моды?

Используй `modes/ru/`, если выполняется хотя бы одно из условий:

- Ты откликаешься преимущественно на **русскоязычные вакансии** (HeadHunter, Habr Career, SuperJob, Работа.ру, корпоративные карьерные страницы)
- Твоё **резюме** на русском или ты переключаешься между RU и EN в зависимости от вакансии
- Тебе нужны ответы и сопроводительные письма на **естественном рус­ском tech-языке**, а не машинный перевод
- Тебе нужно работать со **спецификой российского рынка**: белая/серая зарплата, ДМС как индикатор стабильности, испытательный срок 3 месяца, форматы КТК/ГПХ/ИП

Если большинство твоих вакансий на английском — оставайся на стандартных модах в `modes/`. Английские моды автоматически подхватывают русскоязычные вакансии, когда Claude их распознаёт — но они не знают специфику российского рынка с тем же уровнем детализации.

## Как активировать?

У career-ops нет «языкового переключателя» как code-флага. Вместо этого есть два пути:

### Путь 1 — На сессию, по команде

Скажи Claude в начале сессии:

> "Используй русские моды из `modes/ru/`."

или

> "Оценивай вакансии и пиши отклики на русском — используй `modes/ru/_shared.md` и `modes/ru/vakansiya.md`."

Claude будет читать файлы из этой папки вместо `modes/`.

### Путь 2 — Постоянно, через профиль

Добавь в `config/profile.yml` языковые настройки:

```yaml
language:
  primary: ru
  modes_dir: modes/ru
```

Напомни Claude в первой сессии уважать это поле ("Посмотри в `profile.yml`, я задал `language.modes_dir`"). После этого Claude будет автоматически брать русские моды.

> Примечание: Поле `language.modes_dir` — это соглашение, не жёстко закодированная схема. Если мейнтейнеры захотят структурировать иначе, поле можно переименовать в любой момент.

## Что переведено?

Эта итерация покрывает четыре режима с наибольшим практическим эффектом:

| Файл | Переведён из | Назначение |
|------|-------------|-----------|
| `_shared.md` | `modes/_shared.md` (EN) | Общий контекст, архетипы, глобальные правила, специфика RU-рынка |
| `vakansiya.md` | `modes/oferta.md` (ES) | Полная оценка одной вакансии (Блоки А-F) |
| `otliknutsya.md` | `modes/apply.md` (EN) | Живой ассистент для заполнения форм заявки |
| `pipeline.md` | `modes/pipeline.md` (ES) | Инбокс URL / Second Brain для накопленных вакансий |

Остальные моды (`scan`, `batch`, `pdf`, `tracker`, `auto-pipeline`, `deep`, `contacto`, `ofertas`, `project`, `training`) намеренно не включены. Они продолжают работать через оригиналы EN/ES, поскольку их содержимое — в основном tooling, пути и команды конфигурации — должно оставаться языково-нейтральным.

## Что остаётся на английском?

Намеренно не переводится — это стандартный tech-словарь:

- `cv.md`, `pipeline`, `tracker`, `report`, `score`, `archetype`, `proof point`
- Названия инструментов (`Playwright`, `WebSearch`, `WebFetch`, `Read`, `Write`, `Edit`, `Bash`)
- Значения статусов в трекере (`Evaluated`, `Applied`, `Interview`, `Offer`, `Rejected`, `Discarded`, `SKIP`)
- Фрагменты кода, пути к файлам, команды

Моды используют русский tech-язык, как говорят в реальных engineering-командах в Москве, Питере или Новосибирске: основной текст по-русски, профессиональные термины на английском там, где они общеприняты. Никаких насильственных переводов «pipeline» в «конвейер» или «cv.md» в «резюме.md».

При адаптации или расширении модов придерживайся этого словаря — так тон останется последовательным:

| Skills | Навыки / Компетенции |
| Interview | Интервью / Собеседование |
| Hiring manager | Нанимающий менеджер / Hiring Manager |
| Recruiter | Рекрутер |
| AI | ИИ (Искусственный интеллект) |
| Requirements | Требования |
| Career history | Карьерный путь / Опыт работы |

## Источники для компенсационных исследований

При работе с российскими вакансиями используй:
- **HH.ru Аналитика** — медианные зарплаты по профессиям и регионам
- **Habr Salary** (habr.com/ru/salary/) — зарплаты в IT
- **Glassdoor** — для международных компаний с офисом в РФ
- **Levels.fyi** — для международных tech-компаний

## Вклад в проект

Если хочешь улучшить перевод или перевести дополнительный мод:

1. Открой Issue с предложением (согласно `CONTRIBUTING.md`)
2. Придерживайся словаря выше, чтобы сохранить последовательность тона
3. Переводи осмысленно и идиоматично — никакого дословного перевода слово в слово
4. Сохраняй структурные элементы (Блоки А-F, таблицы, блоки кода, инструкции для инструментов) точно
5. Протестируй с реальной вакансией с HH.ru или Habr Career перед отправкой PR
