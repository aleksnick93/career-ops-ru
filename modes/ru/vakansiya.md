# Режим: vakansiya -- Полная оценка A-F

Когда кандидат вставляет вакансию (текст или URL), ВСЕГДА выдавать 6 блоков:

## Шаг 0 -- Определение архетипа

Классифицировать вакансию по одному из **22 системных архетипов** (см. `modes/ru/_shared.md` → North Star -- Целевые роли). Если гибрид — указать 2 ближайших. Это определяет:
- Какие proof points приоритизировать в блоке B
- Как переписать summary в блоке E
- Какие истории STAR подготовить в блоке F

**ВСЕГДА после определения архетипа** — сверить с `modes/_profile.md` → секция `Personal Filter Overrides`:

1. Если определённый архетип попадает в список **исключённых архетипов** → завершить оценку как `SKIP`:
   - **Не** выполнять блоки B-F
   - **Не** генерировать PDF
   - В `data/applications.md` (через TSV в `batch/tracker-additions/`) — статус `SKIP`, score `0.0/5`, заметка `skipped: archetype #N excluded per _profile.md`
   - В отчёте (если создаётся) — только блок A + причина SKIP
2. Если архетип — **гибрид из исключённого + целевого** (например, Backend + ML components) → **принимать к оценке**, исключение применяется только к чистым попаданиям
3. Если архетип в **целевом списке** (subset из `_profile.md`) → продолжать полную оценку по блокам B-F

Эта проверка экономит токены и защищает кандидата от траты внимания на нерелевантные вакансии.

## Блок A -- Краткое описание роли

Таблица с:
- Определённый архетип (номер + название из `_shared.md`)
- Страна вакансии (RU / BY / KZ / AM / GE / UZ / international)
- Domain (platform/agentic/LLMOps/ML/enterprise/backend/data/и т.д.)
- Функция (build/consult/manage/deploy)
- Уровень (seniority)
- Удалёнка (full/гибрид/офис)
- Размер команды (если указан)
- TL;DR в 1 предложение

**Условная проверка: IT-аккредитация Минцифры**

Применяется **ТОЛЬКО** при выполнении ВСЕХ трёх условий:
1. Страна вакансии = RU
2. `config/profile.yml` → `candidate.gender: male`
3. `modes/_profile.md` → `mobilization_aware: true`

Если условия выполнены → проверить статус компании в реестре аккредитованных IT-организаций Минцифры (`https://www.gosuslugi.ru/itorgs`) и указать: статус (аккредитована / нет / не проверено), льготы для сотрудника (отсрочка от армии, льготная ипотека, пониженные страховые взносы).

В остальных случаях (женщина-кандидат, не-RU вакансия, `mobilization_aware: false` или поле gender не установлено) этот пункт **пропускается без упоминания** — не включать в отчёт вообще.

## Блок D -- Компенсация и спрос

Использовать WebSearch для:
- Актуальные зарплаты по роли — источники по регионам см. `modes/ru/_shared.md` → секция «Компенсационная разведка»
- Репутация компенсации компании
- Тренд спроса на роль

Таблица с данными и указанием источников. Если данных нет — сказать об этом, а не выдумывать.

**Валютный пересчёт — обязательно:**
- Для RUB ↔ USD/EUR — использовать курс ЦБ РФ (`https://www.cbr.ru/currency_base/daily/`) не старше 7 дней
- Для BYN — курс НБ РБ; для KZT — курс НБ РК; для AMD — ЦБ Армении; для GEL — НБ Грузии; для UZS — ЦБ РУз
- Fallback: `https://open.er-api.com/v6/latest/USD`
- **В отчёте фиксировать дату курса** (например: «1 USD = 92.15 RUB, ЦБ РФ от 2026-04-10»)

**Российский рынок -- Обязательные проверки** (применяются к RU / BY / KZ / AM / GE / UZ вакансиям):
- Белая или серая зарплата? (см. opt-in red flags ниже — `gray_salary`, `non_white_salary_kz_by`)
- Gross или net? Пересчитать: net_RU = gross × 0.87 (НДФЛ 13%). Для BY/KZ/AM/GE/UZ — использовать ставки из `config/profile.yml` → `ru_cis_tax.<страна>.income_tax_pct`
- ДМС включён? С какого дня — с выхода или после испытательного?
- Формат: КТК / ГПХ / ИП / Самозанятый? Если ГПХ — рассчитать эквивалент КТК (+30-50% к ставке за отсутствие соцгарантий)
- Годовой бонус / премия? Гарантированная или по KPI?
- Опционы / RSU / ESOP? Проверить vesting, cliff и ликвидность

### Блок D.1 -- Red Flags Workflow (opt-in)

**ВСЕГДА выполнять** после определения зарплатной структуры:

1. **Прочитать** `modes/_profile.md` → секция `Active Red Flags` — только эти флаги будут применяться
2. **Прочитать** `modes/ru/_shared.md` → секция `RU/CIS Red Flags Catalog` — системный каталог с severity и применимостью
3. **Определить страну** вакансии (RU / BY / KZ / AM / GE / UZ / международная) — берётся из Блока A
4. **Для каждого активного флага** из `_profile.md`:
   - Проверить применимость к стране вакансии (колонка «Применимость» в каталоге)
   - Если применим — искать признаки в JD: ключевые слова, отсутствие информации, явные указания
   - Использовать override severity если задан в `_profile.md`, иначе severity по умолчанию из каталога
5. **Специальная проверка `no_minciforming_accreditation`**:
   - Применяется ТОЛЬКО если ВСЕ три условия выполнены (см. Блок A): страна=RU, `gender: male`, `mobilization_aware: true`
   - В остальных случаях — пропустить без упоминания
6. **Специальная проверка `consider_blacklisted`**:
   - Если компания из `portals.yml` → `blocked_employers` И `consider_blacklisted: false` в `_profile.md` → завершить оценку как `SKIP` с пометкой `blacklisted: <reason>`
   - Если `consider_blacklisted: true` → продолжить оценку, но добавить hard flag в таблицу red flags с цитатой источника
7. **Сумма severity** всех найденных флагов = вычет из Global score (Блок F и итоговый score)

## Пост-оценка

**ВСЕГДА** после генерации блоков A-F:

### 1. Сохранить report .md

Сохранить полную оценку в `reports/{###}-{company-slug}-{YYYY-MM-DD}.md`.

- `{###}` = следующий порядковый номер (3 цифры, с ведущими нулями)
- `{company-slug}` = название компании в lowercase, без пробелов (использовать дефисы)
- `{YYYY-MM-DD}` = текущая дата

**Формат report:**

```markdown
# Оценка: {Компания} -- {Роль}

**Дата:** {YYYY-MM-DD}
**Архетип:** {определённый}
**Score:** {X/5}
**URL:** {URL вакансии}
**PDF:** {путь или ожидается}

---

## A) Краткое описание роли
(полное содержание блока A)

## B) Соответствие резюме
(полное содержание блока B)

## C) Уровень и стратегия
(полное содержание блока C)

## D) Компенсация и спрос
(полное содержание блока D)

## E) План персонализации
(полное содержание блока E)

## F) План собеседований

## G) Черновики ответов на форму отклика

---

```

### 2. Зарегистрировать в tracker

**ВСЕГДА** регистрировать в `data/applications.md`:
- Следующий порядковый номер
- Текущая дата
- Компания
- Роль
- Score: средняя оценка match (1-5)
- Статус: `Evaluated`

**Формат tracker:**

